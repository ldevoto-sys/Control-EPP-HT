Plan de Pruebas Manuales — Control de EPP

Documento relacionado: HT-AP-02 · HT-AP-03

1Objetivo y alcance

Verificar los flujos principales por rol antes de liberar la aplicación a producción, mediante un protocolo de prueba manual estructurado que el equipo QA puede completar en una sola sesión.

  • Tiempo estimado: 90–120 minutos en total.
  • Prerequisito: acceso a control-epp-ht-production.up.railway.app con usuarios de prueba por rol (ver sección 2).
  • El administrador debe crear los usuarios de prueba antes de iniciar las pruebas (Módulo A).

3Módulos de prueba

# Acción Resultado esperado OK / NOK
Módulo A — Administración  ·  15 min
A1 Iniciar sesión como administrador Redirige al dashboard con resumen de stock y alertas □
A2 Crear usuario "operador@hidrotecnica.cl" con RUT válido Usuario aparece en la lista □
A3 Intentar crear usuario con RUT inválido Mensaje de error de validación □
A4 Intentar crear usuario con email inválido Mensaje de error de validación □
A5 Crear usuarios para los 4 roles restantes Todos aparecen en la lista □
A6 Filtrar/ordenar lista de usuarios Lista se actualiza correctamente □
A7 Editar un usuario (cambiar nombre) Cambio se refleja en la lista □
A8 Ver catálogo de EPP Lista de EPP con stock disponible □
A9 Crear un EPP nuevo (ej. "Guante térmico") Aparece en el catálogo □
A10 Intentar eliminar EPP con stock asociado Mensaje de error 409 □
Módulo B — Operador: Solicitud  ·  15 min
B1 Iniciar sesión como operador Dashboard de operador visible □
B2 Crear nueva solicitud con 2 ítems de EPP Solicitud creada en estado "pendiente" □
B3 Ver "Mis solicitudes" Solicitud aparece con estado y número □
B4 Intentar crear solicitud sin seleccionar EPP Formulario no se envía (validación) □
Módulo C — Autorizador: Revisión  ·  10 min
C1 Iniciar sesión como autorizador Lista de solicitudes pendientes visible □
C2 Abrir la solicitud creada en B2 Detalle con ítems y stock disponible visible □
C3 Aprobar la solicitud Estado cambia a "aprobada"; operador puede verlo □
C4 Crear segunda solicitud (desde operador) y rechazarla Estado cambia a "rechazada" con motivo □
Módulo D — Bodega: Entrega  ·  20 min
D1 Iniciar sesión como bodega Lista de entregas pendientes visible □
D2 Ver solicitud aprobada (de C3) en "Pendientes de entrega" Aparece con número e item_count □
D3 Registrar entrega de la solicitud Entrega guardada; solicitud desaparece de pendientes □
D4 Realizar entrega directa (sin solicitud) con 1 ítem Entrega directa registrada □
D5 Ingresar stock de un EPP (registrar ingreso) Historial de movimientos muestra el ingreso con usuario □
D6 Ver stock general Tabla con epp_id, epp_nombre y cantidades correctas □
D7 Registrar una devolución de EPP Devolución registrada; stock sube □
Módulo E — Consulta / Reportes  ·  15 min
E1 Iniciar sesión como consulta Menú de consulta visible □
E2 Buscar trabajador por nombre Resultado con asignaciones actuales □
E3 Ver detalle de asignación Lista de EPP asignado + historial de entregas □
E4 Ver documentación / matriz de asignaciones Tabla con trabajadores y EPP □
E5 Iniciar sesión como admin, ir a Reportes Dashboard sin error (no pantalla en blanco) □
E6 Verificar reporte "Stock crítico" Muestra EPP bajo mínimo (total + items) □
E7 Verificar reporte "Vencimientos" Muestra EPP próximos a vencer □
E8 Descargar reporte en PDF PDF descargado sin error □
Módulo F — Seguridad y accesos  ·  10 min
F1 Intentar acceder a URL de admin como operador Redirige a /dashboard o muestra "sin acceso" □
F2 Cerrar sesión Redirige a login; no accede con botón Atrás □
F3 Cambiar contraseña del usuario actual Nuevo password funciona en el próximo login □
F4 Ingresar contraseña actual incorrecta al cambiar Mensaje de error □

5Criterio de aprobación

Verde 37/37 casos aprobados → apto para producción.
Amarillo Hasta 3 NOK menores que no bloquean flujos críticos → liberar con observaciones registradas.
Rojo Cualquier NOK en módulos B, C o D (flujo principal de solicitud y entrega) → no liberar.
